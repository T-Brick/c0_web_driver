About

C0deine is a compiler for c0. It is written in Lean 4, which allows us to express the formal semantics in the same language as the compiler itself. We hope to fully verify the entire compiler at some point/soon.

C0deine implements a number of "sub"-languages of C0 as well as fixing some bugs in the existing compiler. See this document for information about the languages themselves, as well as a list of changes/corrections. If you find any issues, please report them here.

Verification Status

Currently the following parts of the compiler are verified:
  • Abstract Syntax Tree: Static and Dynamic Semantics are formalised.
  • Typechecker: Output is verified to be a well-typed program where all variables are initialised before use and all control-flow paths return. Though, the output hasn't been verified to be correct with respect to the dynamic semantics.
  • Typed Syntax Tree: Static Semantics are definitionally enforced.
  • WebAssembly: Static Semantics are formalised; Dynamic Semantics are partially formalised.
Hopefully more will be finished soon! : )

Aside about using Lean

One thing striking about learning Lean was how well Standard ML and OCaml could be translated into it. From what I've seen, if I have a working function in SML/OCaml, all I really need to do is change the keywords (and annotate the function args/return type) and it generally just works without fuss. Even proofs about SML code can (in my experience) be one-to-one translated into the same proof about the Lean version of the code.

There's a lot more about Lean that I like (how readable it is, tooling, meta-programming, etc.) but I won't go into more depth here. In general, I would like to make it clear that Lean markets itself as a "programming language and theorem prover", but the "theorem proving" part is largely optional. And when evaluating it as a programming language, I struggle to find a reason, other than legacy code and perhaps more mature libraries, to use SML or OCaml over Lean.
